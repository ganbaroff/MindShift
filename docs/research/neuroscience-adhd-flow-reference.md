# Нейробиология СДВГ и состояния потока — Референс для MindFocus

> Источник: «Нейробиологические основы фокуса, дефицита внимания и состояния потока»
> Сохранено для использования в архитектурных решениях и bolt-спеках

---

## 1. Нейрохимия СДВГ → прямые следствия для UX

### Дофаминовый дефицит
- DAT1-ген → быстрый обратный захват дофамина → хронически низкий базальный уровень
- Мозг в состоянии «голода по стимуляции» → компульсивный скроллинг, переключение вкладок
- **Вывод:** каждое завершённое действие должно давать немедленное дофаминовое вознаграждение (микроанимация, тактильный фидбек)

### Норэпинефриновый дефицит
- ПФК не фильтрует фоновый «шум» — ни внешний, ни внутренний
- Поддержание фокуса стоит колоссальных энергетических затрат
- **Вывод:** UI обязан убирать визуальный шум агрессивнее, чем для нейротипичного дизайна

### Серотонин (81% исследований)
- Дисбаланс → эмоциональная дисрегуляция, импульсивность, тревожность
- **Вывод:** любой стрессогенный паттерн в интерфейсе действует сильнее, чем кажется разработчику

## 2. Паралич задач и энергия активации

**Нейробиология:** Dopamine Transfer Deficit (2011) — дофаминовые нейроны при СДВГ не реагируют на упреждающие сигналы вознаграждения, только на немедленное. Базальные ганглии не преодолевают порог активации для задач с отложенным результатом.

**Следствия для UI:**
- Каждая задача должна начинаться с мгновенного, видимого прогресса (первый шаг < 2 мин)
- «Правило 2 минут» и «5-минутные спринты» эффективнее Pomodoro (25 мин = слишком высокий барьер)
- Формулировка: «Давай поработаем над этим ровно 5 минут» → преодоление инерции
- AI-декомпозиция крупных задач — прямая компенсация дефицита инициации

## 3. RSD — Дисфория чувствительности к отвержению

**Нейробиология:** гиперактивная амигдала + ослабленный кортикальный тормоз → любой сигнал об ошибке активирует «бей/беги/замри».

**Критические паттерны, которых нельзя делать:**
| Паттерн | Реакция СДВГ-мозга |
|---|---|
| Красный счётчик просроченных задач | Воспринимается как личный упрёк → желание удалить приложение |
| Обнуление серии (streak) | Спираль стыда → полное прекращение использования |
| «Сессия истекла», «Данные введены неверно» | Воспринимается как личная критика |
| Таймеры обратного отсчёта | Перманентная бдительность → когнитивное истощение |

**Что делать:**
- Red → Grey/Pastel для любых статусных индикаторов
- Парадигма «предложение помощи» вместо «указание на ошибку»
- Принцип «Forgiveness & Recovery» — поощрение за возвращение, не за непрерывность
- Пользователь контролирует все badge counts (on/off)

## 4. Состояние потока — 4 фазы и таймер

| Фаза | Длительность | Нейрохимия | Требование к UI |
|---|---|---|---|
| **Борьба** | 10–15 мин | Кортизол + норэпинефрин, загрузка контекста | Полная изоляция от триггеров. Любое прерывание = возврат к нулю |
| **Отпускание** | ~5 мин | Оксид азота, снижение симпатического тонуса | Не требовать агрессивных действий |
| **Поток** | 60–90 мин | Транзиторная гипофронтальность, дофамин + анандамид + эндорфины | UI должен «исчезнуть» — только рабочая среда |
| **Восстановление** | 10–30 мин | Серотонин + окситоцин | Мягкое завершение, итоги, блок новых сложных задач |

**Ключевые цифры:**
- Глубокий поток: 60–90 мин
- Суточный лимит потока: ~4 часа
- Стоимость прерывания: **23 мин 15 сек** на восстановление (UC Irvine)
- Уведомления каждые 15–20 мин = физически невозможно войти в поток

**Условия входа (Чиксентмихайи):**
1. Чёткие цели (убрать неопределённость)
2. Немедленная однозначная обратная связь
3. Баланс «сложность задачи = текущие навыки»

## 5. Гиперфокус vs Поток — критическое различие

| | Поток | Гиперфокус (СДВГ) |
|---|---|---|
| Триггер | Оптимальный баланс сложности | Новизна, немедленное вознаграждение, кризис |
| Контроль | Можно сознательно выйти | Непроизвольное «залипание» |
| ЭЭГ-паттерн | Норма | Патологически низкие альфа и бета |
| После | Позитивный аффект | Истощение, перцептивная слепота |

**Вывод для UI:** приложение должно быть «внешним тормозом» — напоминать о перерыве, питье воды, смене позы. Не использовать бесконечный скроллинг / автовоспроизведение для индукции гиперфокуса.

## 6. Слепота ко времени — визуальные таймеры

**Нейробиология:** ПФК + базальные ганглии + мозжечок = «биологические часы». Дофаминергическая дисфункция → десинхронизация. Время существует в двух модальностях: «Прямо сейчас» и «Не сейчас».

**Цифровые часы (14:45) — неэффективны:** требуют когнитивной конверсии абстрактных символов → перегружают рабочую память.

**Time Timer — клинически обоснован:**
- Клинические испытания (Gunther et al., 2019):
  - TPA (Time Processing Ability): p=0.019 (значимое улучшение)
  - DTM (навыки управления временем): p=0.01
- Механизм: уменьшающийся цветной диск/сектор → пространственно-визуальное восприятие количества времени
- Периферический взгляд достаточен — нет дополнительной когнитивной нагрузки

**Применение в MindFocus (Bolt 5.2):**
- Focus Timer = убывающий визуальный диск, а не цифровой обратный отсчёт
- Прогресс через убывающие цветные блоки / анимации заполнения

## 7. Поведенческие сигналы — пассивное профилирование

| Сигнал | Корреляция с СДВГ |
|---|---|
| Медленный/вариативный отклик на уведомления | p=0.047, d=1.05 / p=0.010, d=0.84 |
| Высокая энтропия клавиатуры (Backspace, паузы) | p=0.005, коррелирует с дефицитом исполнительной функции |
| Чередование секундных открытий с многочасовым зависанием | Маркер гиперфокуса или когнитивной перегрузки |
| Хаотичные паттерны переключения экранов | Когнитивная дисрегуляция |

**Применение:** при накоплении этих сигналов → адаптивный UI:
- Скрыть второстепенную навигацию
- Отключить анимации
- Заблокировать пуш-уведомления
- Перейти в режим минимальной когнитивной нагрузки

## 8. Scaffolding — UI как экзоскелет исполнительных функций

| Элемент | Цель | Реализация |
|---|---|---|
| Видимость состояния | Компенсация дефицита рабочей памяти | Вся нужная инфо на одном экране, никаких скрытых меню |
| Визуальный таймер | Обход слепоты ко времени | Time Timer, не цифровой отсчёт |
| Микровознаграждение за каждый шаг | Стимуляция дофамина | Haptics, конфетти, мгновенное подтверждение |
| Изоляция стимулов | Снижение шума (норэпинефрин) | White space, Focus Mode, скрытие боковых панелей |
| AI-декомпозиция | Снижение энергии активации | Разбить «Организовать переезд» на 5-минутные шаги |

## 9. Chunking — Правило трёх

- 7±2 Миллера неактуально при СДВГ — реальный порог значительно ниже
- **Правило трёх** — безопасный лимит: 3-4 элемента в блоке
- Список из 10–15 задач гарантирует сбой исполнительной функции
- **Progressive disclosure:** следующий блок из 3 шагов появляется только после завершения предыдущего

**Форматы подачи:**
- Текст + иконка + цвет одновременно (обогащение рабочей памяти)
- Многоканальность снижает когнитивную перегрузку

## 10. Цветовые правила — физиологический уровень

| Цвет | Реакция СДВГ-мозга | Применение |
|---|---|---|
| Красный (счётчики, бейджи) | Выброс кортизола, salience bias, urgency bias | ❌ Запрещён для статусов ошибки/просрочки |
| Пастельные тона | Снижение симпатического возбуждения | ✅ Статусные индикаторы |
| Глубокий синий, зелёный | Ассоциация с безопасностью | ✅ Приоритетные цвета интерфейса |
