# Focus Audio — Нейробиология звука и UX-архитектура

> Источник: «Доказательный нейробиологический и UX-анализ влияния звука на когнитивные функции»
> Прямое применение: **Bolt 5.2 — Focus Audio (Web Audio API)**

---

## 1. Цветовые шумы — что включать, что запрещать

### Таблица применимости

| Шум | Спектр | Для кого работает | Решение для MindFocus |
|---|---|---|---|
| **Розовый** | -3 дБ/окт (1/f) | ✅ Все пользователи | **Умный дефолт** — безопасен для всех |
| **Коричневый** | -6 дБ/окт (1/f²) | ✅ ADHD, выгорание, brain fog | **ADHD-режим** + Recovery Break |
| **Белый** | 0 дБ/окт (плоский) | ⚠️ Только невнимательный тип ADHD | Опция, не дефолт |
| **Синий** | +3 дБ/окт | ❌ Никому | **Исключить полностью** |

### Ключевые данные

- Белый/розовый шум для ADHD: **g=0.249** (p<0.0001) — значимое улучшение когниции
- Белый/розовый шум для нейротипиков: **g=−0.212** (p=0.0036) — значимое ухудшение
- Неконтролируемый низкочастотный шум (LFN): **SMD=−0.37** — угнетает высшие когнитивные функции
- **Обязателен High-pass фильтр** на всех треках (ниже 60 Гц срезать)

### Нейробиологическое объяснение (критично для дизайна)

**Стохастический резонанс** — для ADHD-мозга:
- ADHD = гиповозбуждение (избыток тета/альфа на ЭЭГ, дефицит дофамина)
- Внешний шум → повышает фоновое возбуждение → «поднимает» когнитивный сигнал выше порога
- Белый шум = «нейрологический протез» для ADHD-мозга

**ВАЖНО:** U-образная кривая:
- Гиперактивный тип ADHD → белый шум может перевозбудить → переключить на коричневый
- Коричневый шум = «успокоение» мозга, подавление апериодической активности
- Парадигма 2026: не «стимулировать» фокус, а «успокоить» мозг чтобы фокус проявился сам

### Что означает «коричневый шум» для Bolt 5.2

```
Коричневый шум ≠ низкочастотный гул
Коричневый шум = глубокое, насыщенное звучание (океан, мощный водопад)
Без резких высокочастотных «шипящих» компонентов белого шума
Эффект: стабилизирует мозговую активность, минимизирует скачки дофамина/глутамата
```

## 2. Бинауральные ритмы — точные технические параметры

### Физика (для Web Audio API)

```
Левое ухо: тон на частоте F (Гц)
Правое ухо: тон на частоте F + Δ (Гц)
Мозг воспринимает: иллюзорный ритм с частотой Δ Гц

Технические требования:
- Несущая частота: 400–500 Гц (оптимум)
- Разница частот (Δ): не более 35 Гц (иначе иллюзия распадается)
- ТОЛЬКО стереонаушники (монодинамик = нет эффекта)
- Наложение на шум (не на музыку) — сохраняет FFR-ответ
```

### Три пресета для MindFocus

| Пресет | Частота Δ | Состояние | Когда применять |
|---|---|---|---|
| **Deep Focus** | 14–40 Гц (Бета/Гамма) | Пиковая концентрация, рабочая память | Во время фокус-сессии |
| **Calm Flow** | 8–14 Гц (Альфа) | Спокойная концентрация без напряжения | Начало сессии, лёгкие задачи |
| **Recovery Break** | 4–8 Гц (Тета) | Снижение тревожности, релаксация | Break-время, не во время работы |

### Критические требования к безопасности

1. **Эпилепсия = абсолютное противопоказание**
   - При первой активации бинауральных ритмов → блокирующий Modal (нельзя закрыть без подтверждения)
   - Текст: «Бинауральные ритмы противопоказаны при эпилепсии и судорожных синдромах»
   - Checkbox: «У меня нет эпилепсии в анамнезе»

2. **Проверка наушников**
   - Запрашивать подтверждение что пользователь в стереонаушниках
   - Через API устройства если возможно

3. **Pre-roll — 5–10 минут до начала задачи**
   - Ритмы должны начаться ДО фактической работы
   - Мозгу нужна фаза индукции — менее 9-10 мин = нет измеримого эффекта

### Метаанализ эффективности

- Общий эффект на когниции: **g=0.45** (статистически значимый, средний)
- Бета/Гамма: 10 величин эффекта на память, решение задач, бдительность
- Тета: эффективно для тревожности, НО — негативно для рабочей памяти

## 3. Lo-Fi как оптимальный музыкальный фон

### Почему Lo-Fi работает (нейробиология)

- DMN (Default Mode Network) при тишине → начинает «охотиться» за стимуляцией → прокрастинация
- Lo-Fi «заполняет» ровно столько когнитивного пространства, чтобы DMN не генерировала внутренние отвлечения
- 70–90 BPM ≈ частота сердцебиения в покое → фоновый успокоительный эффект
- Нет вокала = нет обработки семантической информации = нет конкуренции с рабочей памятью
- Ностальгический «тёплый» звук (потрескивание винила) → положительный аффект без возбуждения

### «Эффект Моцарта» — развенчан

- Классика не улучшает когниции сама по себе
- Эффект объяснялся: контрольная группа сидела в некомфортной тишине → снижение тонуса
- Любая любимая музыка даёт аналогичный эффект через дофамин (nucleus accumbens)
- НО: музыка с вокалом → конкуренция рабочей памяти → вред для сложных задач

### Параметры для Lo-Fi плеера

```
Темп: 70–90 BPM
Громкость: 45–65 дБ (хард-лимитер)
Структура: консонантные джазовые гармонии, нет резких перкуссионных переходов
Без вокала (обязательно)
Loop-based (зацикленные) — предсказуемость = нет сюрпризов для мозга
```

## 4. Природные звуки — для перерывов (ART)

### Теория восстановления внимания (ART)

- Направленное внимание = **ограниченный физиологический ресурс**
- Природные звуки → «мягкое вовлечение» (soft fascination) → непроизвольное внимание
- Непроизвольное внимание НЕ истощает ресурс, а восстанавливает его

### Нейровизуализация: природа vs урбанизм

| Стимул | Эффект на мозг |
|---|---|
| Природные звуки | ↑ функциональная связность слуховых сетей + соматомоторных областей |
| Урбанистический шум | ↑ энтропия BOLD-сигнала в posterior cingulate (конфликт, стресс) |

### Фрактальность = нейронный резонанс

- Природные звуки имеют спектр 1/f или 1/f² (розовый/коричневый шум)
- Мозг сам работает с 1/fβ динамикой
- Совпадение внутреннего и внешнего спектра → минимизация «ошибки предсказания» (predictive coding)
- Мозг воспринимает как «экологически безопасная среда» → падение кортизола

### Применение в MindFocus

- **Во время Break** (Recovery Break) → автоматически переключать на природные звуки
- Деактивировать бета-бинауральные ритмы при переходе на отдых
- Активировать тета-ритмы (4-8 Гц) + природные звуки = максимальное восстановление

## 5. UX-архитектура звука — точные технические параметры

### Sonic Anchor (Звуковой якорь)

```
Что: уникальный 2-сек тональный сигнал при старте таймера
Цель: павловское обусловливание — мозг учится: «этот звук = начало фокуса»
Эффект: через 10–15 сессий сигнал сам вызывает рефлекторное торможение DMN
Требования:
  - Мягкий, не пугающий (синтезатор / тибетская чаша / перезвон)
  - Уникальный — не используется нигде кроме старта фокус-сессии
  - Постоянный (всегда один и тот же звук для этого пользователя)
```

### Аудио-переходы (Fade)

```
Тип кривой: экспоненциальная или S-образная логарифмическая
Длительность: ровно 500 мс
НЕ линейная (линейная создаёт иллюзию «внезапного схлопывания»)

Web Audio API реализация:
gainNode.gain.setTargetAtTime(0, audioCtx.currentTime, 0.1); // экспоненциальный decay
// tau = 0.1 → -3dB через 0.1 сек, -60dB через ~0.7 сек = плавное
```

### Управление громкостью (Gain Staging)

| Уровень (дБ) | Эффект |
|---|---|
| 45 дБ | Оптимум: удержание внимания, снижение стресса |
| 60–65 дБ | Допустимо: кратковременное улучшение рабочей памяти |
| 65+ дБ | Рост маркеров физиологического стресса |
| 85+ дБ | Деградация внимания, патологическое изменение альфа/бета ЭЭГ |

**Вывод: хард-лимитер на 65 дБ в Web Audio API — обязательно.**

### Тайминг звуковой хореографии

| Фаза | Аудио-действие |
|---|---|
| За 5 мин до сессии | Pre-roll бинауральных ритмов (индукция) |
| Старт таймера | Sonic Anchor (2 сек) |
| Борьба (0–15 мин) | Коричневый/розовый шум + Lo-Fi |
| Поток (15–90 мин) | Шум продолжается, бета/гамма бинауральные |
| За 60 сек до конца | Предупреждающий мягкий тон (67-74 мс реакция) |
| Конец → Break | 500 мс fade-out рабочего звука |
| Break | Fade-in природных звуков + тета-ритмы |
| Конец Break | 500 мс fade-out → Sonic Anchor → новая сессия |

### Скрытое когнитивное слепое пятно (Metacognitive Bias)

> Пользователи **переоценивают** свою продуктивность под любимую музыку.
> «Мне приятно» ≠ «Я продуктивен».

Импликация для UX: не давать пользователю полностью переключиться на вокальную музыку во время сложных задач. Или как минимум — предупреждение.

## 6. Умный дефолт vs Персонализация

### Smart Default (критично — 95% пользователей не меняют настройки)

```
Дефолт при онбординге:
- Розовый шум (умеренная громкость, 50 дБ)
- Или Lo-Fi без вокала (70-80 BPM)

НЕ дефолт:
- Белый шум (плохо для нейротипиков)
- Синий шум (исключить)
- Бинауральные ритмы (требуют дисклеймер)
```

### Power User персонализация (для конверсии в Pro)

```
Soundscape Mixer — до 3 слоёв одновременно:
1. Музыкальная основа (Lo-Fi треки)
2. Маскировочный слой (природный шум / цветной шум)
3. Нейромодулирующий слой (бинауральные ритмы)

Каждый слой = независимый volume fader
Сохранение кастомных пресетов → Pro-фича
```

### ADHD Mode — алгоритм выбора

```javascript
// Псевдокод логики ADHD Mode
if (user.adhdMode && user.currentState === "brain_fog") {
  audio.primary = "brown_noise"; // успокоение, подавление апериодической активности
} else if (user.adhdMode && user.currentState === "hyperactive") {
  audio.primary = "brown_noise"; // НЕ белый — не перевозбуждать
} else if (user.adhdMode && user.currentState === "inattentive") {
  audio.primary = "white_noise"; // стохастический резонанс
} else {
  audio.primary = "pink_noise"; // дефолт для всех
}
```

## 7. Прямая связь с Bolt 5.2 — Focus Audio

### Web Audio API архитектура

```
AudioContext
├── OscillatorNode (L, 440 Hz) ─┐
│                               ├→ StereoPanner → GainNode → destination
├── OscillatorNode (R, 440+Δ Hz)┘    (binaurale)    (volume)
│
├── AudioBufferSourceNode (noise: pink/brown/white)
│   └── BiquadFilterNode (high-pass 60 Hz) → GainNode → destination
│
└── AudioBufferSourceNode (lo-fi / nature)
    └── GainNode → destination
```

### Что реализовать в Bolt 5.2

| Компонент | Приоритет | Техника |
|---|---|---|
| Розовый шум (дефолт) | 🔴 Must | Web Audio API noise generation |
| Коричневый шум | 🔴 Must | Web Audio API, +Brownian filter |
| High-pass фильтр 60 Гц | 🔴 Must | BiquadFilterNode (highpass, 60 Hz) |
| Хард-лимитер 65 дБ | 🔴 Must | GainNode + DynamicsCompressorNode |
| 500 мс логарифм. fade | 🔴 Must | gain.setTargetAtTime() |
| Sonic Anchor (2 сек) | 🟠 Important | OscillatorNode или AudioBuffer |
| Белый шум (опция) | 🟠 Important | Web Audio API |
| Lo-Fi плеер | 🟠 Important | AudioBufferSourceNode + loop |
| Природные звуки (break) | 🟠 Important | Аудиофайлы в /public |
| Бинауральные ритмы | 🟡 Pro | OscillatorNode L+R с Δ, + дисклеймер |
| Soundscape Mixer (3 слоя) | 🟡 Pro | Сложнее — пост-MVP |
| Pre-roll (5 мин до) | 🟡 Nice | Timer + audio start offset |

### Обязательные UX-требования

1. Звуковой якорь при старте таймера — всегда
2. 500 мс логарифмический fade на всех переходах — всегда
3. Дисклеймер при активации бинауральных — всегда
4. Природные звуки автоматически при переходе в Break — всегда
5. Деактивация всех уведомлений во время Focus — всегда (из плана Bolt 5.2)

### BufferBlock hook (из Bolt 5.1)

```jsx
// Уже готово в TimelineView.jsx:
data-buffer-slot="true"
data-start={item.scheduledStart.toISOString()}
data-end={item.scheduledEnd.toISOString()}

// Bolt 5.2 Focus Audio прикрепляется к этим элементам:
// При наступлении data-start → запустить audio для данного слота
// При наступлении data-end → запустить nature sounds для перерыва
```
