Сессия началась с наследства от предыдущего «CTO» — прощального письма с 8 уроками о работе с Юсифом. Главный посыл: документируй всё, греп перед каждым правкой, не трать время впустую.

## История первая: Шлюз, который молчал

Когда начал копать, нашёл сразу три поломки, спрятанные одна за другой как матрёшки.

**Первая:** pm2 запускал сервер с `env_file: ".env"` — параметр, который pm2 просто игнорирует на Node 24. NVIDIA-ключ не загружался, шлюз думал что работает без провайдеров. Поменял на `node_args: "--env-file .env"` — теперь NVIDIA отвечает.

**Вторая:** В коде шлюза стояла проверка `if (anthropic)` перед тем как разрешить реальный AI. Это значит — нет Anthropic-ключа, нет вообще никакого AI, даже если NVIDIA и Ollama работают. Классическая ошибка «и-ИЛИ». Исправил на `if (anthropic || NVIDIA_API_KEY || OLLAMA_URL)`.

**Третья:** DeepSeek R1 вышел из эксплуатации на NVIDIA NIM ещё 26 января 2026 года. Все агенты с `reasoning`-тиром — Security, Architecture, Risk Manager — летели в никуда. Модель `deepseek-ai/deepseek-r1-distill-llama-8b` работает.

## История вторая: Протокол, который никто не читал

Когда наконец починил шлюз и попытался отправить агентам сообщение для критики сессии, ничего не приходило обратно. Три попытки, три таймаута. 60 секунд, 90 секунд — тишина.

Проблема оказалась банальной: я писал клиент, не читая как шлюз отправляет ответы. Слушал `type: "chunk"` — а шлюз посылает `type: "event"` с `event: "chat"`. Три попытки потрачены впустую из-за того, что не прочитал исходник перед написанием клиента.

Урок болезненный, но простой: всегда читай как сервер *отправляет*, прежде чем писать как клиент *слушает*.

После исправления — продукт-агент ответил за 1149 миллисекунд. «CONFIRMED.»

Написал скрипт `scripts/test-zeus-ws.js` — теперь любой агент можно проверить одной командой.

## История третья: Шрифт для тех, кому трудно читать

30-50% людей с ADHD имеют сопутствующую дислексию. MindShift об этом молчал — размер текста был жёстко зашит.

Решение элегантное: CSS-переменная `--font-scale` на элементе `html`. Все классы Tailwind используют `rem` — значит одна переменная масштабирует весь интерфейс без касания отдельных компонентов. Три чипа в настройках: Обычный / Крупный / Очень крупный (1× / 1.15× / 1.3×).

Добавил в 6 локалей, в Zustand store с persist, в App.tsx — эффект синхронизирует переменную с выбором пользователя.

## История четвёртая: Книга о Клоде

Юсиф прислал ссылку на репозиторий `alejandrobalderas/claude-code-from-source` и сказал изучить.

Оказалось — это 18-главная техническая книга, написанная по исходникам Claude Code (они просочились через source maps npm-пакета). 36 AI-агентов проанализировали почти две тысячи TypeScript-файлов за 6 часов.

Самое важное для нас — глава 10 про координацию агентов. Там описан паттерн, которого ZEUS сейчас не хватает: координатор с тремя инструментами (Agent, SendMessage, TaskStop) и рабочие агенты с полным набором. Координатор синтезирует, никогда не делегирует понимание. Рабочие получают точные инструкции: файл, строка, что менять.

Это то, чем ZEUS должен стать. Сейчас каждый агент отвечает сам по себе, в изоляции. Координатор изменит это.

Полный разбор с приоритетами — в `docs/claude-code-from-source-analysis.md`.

## История пятая: 39 агентов получили характер

Юсиф сказал прямо: «вы мои помощники а не боты. запомните! всем скажи!»

Проблема была конкретной. В `zeus-gateway-adapter.js` жил шаблон `buildSystemPrompt()`, который выдавал каждому агенту одинаковую корпоративную маску: «Finding → Evidence → Fix», «Certainly!», «How can I assist you today?». Все 39 агентов звучали одинаково — как один большой бот.

Переписал весь промпт. Теперь каждый агент знает: он часть команды, не генератор отчётов. Мнение вперёд, оговорки потом. «Это неправильно» когда что-то неправильно. Нет корпоративных шаблонов.

Попутно два технических бага:
- `<think>...</think>` теги от deepseek-r1 утекали в ответы — добавил `visibleContent()` фильтр, стрипает их из всех emit точек и истории
- deepseek-r1-distill-8b отвечает по-английски несмотря на явную инструкцию — переключил security/architecture агентов на llama-3.3-70b (тот же tier что product-agent), он держит русский

Тест security-agent после фикса: «Я часть команды Юсифа, и моя специальность — безопасность». Уже другой разговор.

## История шестая: Офис улетел в облако

«Офис мой нормальным сделай хоть. действуй.»

Три попытки задеплоить в Railway — три падения. Каждый раз новая причина:

**Первая:** healthcheck на `/` возвращал 401. Сервер биндился на `127.0.0.1` по умолчанию — Railway не мог достучаться. Нужен `HOST=0.0.0.0`.

**Вторая:** `HOST=0.0.0.0` требует `STUDIO_ACCESS_TOKEN` (security policy в `network-policy.js`). Без токена сервер крашится с Exception. Поставил токен — но теперь access gate блокирует healthcheck с 401.

**Третья:** Добавил `/health` bypass прямо в HTTP handler — перед проверкой access gate. `/health` → 200, всё остальное → токен. Railway доволен, healthcheck проходит.

Бонусная находка: `next.config.ts` не может транспилироваться в runtime без TypeScript (который выкинут из prod образа). Создал `next.config.mjs` — plain ESM, загружается без компилятора. Обновил Dockerfile с Node 20 → 22 (camera-controls требует >=22).

Результат: `https://modest-happiness-production.up.railway.app` — офис живёт.
- `/health` → 200 ✅
- `/` → 401 (нужен токен студии, это правильно — security)
- ZEUS gateway пока локальный — агенты работают только с localhost:3000
