### TD-008 — ngrok wildcard in production CORS
**Type:** Code debt (security)
**Score:** (2 + 3) × (6 - 1) = **25**

`isNgrokOrigin()` in `_shared/cors.ts` allows any `*.ngrok-free.dev` domain in all environments.
Should be gated to development only.

**Fix:**
```typescript
function isNgrokOrigin(origin: string): boolean {
  if (Deno.env.get('ENVIRONMENT') === 'production') return false
  return /^https:\/\/[a-z0-9-]+\.ngrok-free\.dev$/.test(origin)
}
```
**Effort:** 10 minutes

### TD-010 — Offline queue uses `upsert` without conflict column validation
**Type:** Code debt (correctness)
**Score:** (2 + 3) × (6 - 2) = **20**

`flushQueue` upserts to arbitrary `item.table` with `onConflict: 'id'`. If a queued item's
table doesn't have an `id` primary key, the upsert silently fails (or errors). No table
allowlist enforced in the queue.

**Fix:** Add a `ALLOWED_TABLES` allowlist in offlineQueue.ts; validate on `enqueue()`.
**Effort:** 30 minutes
