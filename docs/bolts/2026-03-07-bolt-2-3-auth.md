## Architecture

### `useAuth` hook
- Lives in `shared/hooks/` — cross-cutting, consumed by App root `mindflow.jsx`
- Single `onAuthStateChange` subscription (unsubscribed on unmount via `subRef`)
- `getSession()` called on mount to avoid auth-screen flash on refresh
- Exposes: `{ user, authLoading, signIn, signUp, signOut }`
- All Supabase access goes through `waitForSupabase()` (single entry point rule)

### Protected routes (mindflow.jsx)
```
onboarding (lang, welcome) → authLoading spinner → !user → AuthScreen → app
```
- `authLoading` prevents the flash of AuthScreen on refresh when session exists
- No "skip" or "continue as guest" — auth is required (daily_tasks need user_id)

### AppBar
- Thin bar above screen content: truncated email (left) + "Sign out" button (right)
- Only shown when `user` is truthy (inside protected route guard)
- Touch target: `minHeight: 30` (not a primary action, so 30px acceptable)

### AuthScreen props
```js
<AuthScreen lang={lang} onSignIn={signIn} onSignUp={signUp} />
```
- View receives functions, never imports Supabase directly
- Error handled by catching prop throws and mapping to neutral strings

## Security Notes (codelably-security-review)

- Passwords never stored in React state beyond the controlled input
- No password logged via `logError` (errors logged without password fields)
- Supabase SDK manages JWT in `localStorage` automatically
- `signOut` calls `sb.auth.signOut()` — revokes server-side session
- `.env.example` documents all required environment variables with security warnings
- `VITE_ANTHROPIC_API_KEY` warning: client-side exposure noted in `.env.example`
